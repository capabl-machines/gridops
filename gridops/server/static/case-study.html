GridOps

An environment-first microgrid operator: train a small model to read hourly grid state, emit valid actions, and improve resilience across normal, heatwave, and crisis conditions.

Open the live environment See measured results

The operating problem

Microgrids are not text tasks. They are constrained control systems.

A useful AI operator has to balance demand, solar, battery state of charge, diesel fuel, tariffs, and outage risk one hour at a time. The output cannot be a paragraph of advice. It must be a bounded action that the environment can execute and score.

GridOps turns that into a training harness: observations in, JSON action out, physics-based feedback, and holdout evaluation against do-nothing and oracle policies.

How we built it

Environment, curriculum, model, evals.

We kept the OpenEnv contract stable and moved training into a separate harness. That made the demo lightweight while preserving a reproducible path from rollout data to a trained action model.

01

OpenEnv runtime

FastAPI and WebSocket endpoints expose reset, step, schema, and dashboard behavior without changing GridOps physics.

02

Curriculum traces

1,418 traces mix deterministic oracle rollouts with teacher-generated edge cases for normal, heatwave, and crisis tasks.

03

SFT action model

Qwen2.5-3B-Instruct was trained with QLoRA to emit only JSON actions for battery, diesel, and demand shedding.

04

Holdout gates

We measured valid action rate, score by task, blackout kWh, battery throughput, and comparison to baselines.

Training contract

The model learned an executable control language.

The target completion is intentionally small: {"battery_dispatch": ..., "diesel_dispatch": ..., "demand_shedding": ...}. This keeps the first milestone focused on reliability and operational score before adding reinforcement learning.

The trained model beats do-nothing across all tasks.

Impact lens

This is a delivery pattern, not just a demo.

For Capabl Machines, GridOps shows a repeatable way to approach operational AI: define the simulator, generate a curriculum, train a compact model, prove it with environment metrics, and keep the runtime deployable.

The current model is intentionally SFT-only. RL is the next phase once the action model is stable enough to optimize directly against cost, outage resilience, fuel discipline, and battery health.
